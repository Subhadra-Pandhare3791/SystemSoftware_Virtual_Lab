          1.Introduction of Language Processor

      *Semantic Gap:

            Language processing activities arise due to the difference between the manner in which a software designer describes the ideas concerning the behavior of software and the manner in which these ideas are implemented in a software system. The designer expresses the ideas in terms related to the application domain & to implement those ideas, their description has to be interpreted in terms related to the execution domain of a computer system We use the term semantics to represent the rules of meaning of a domain and the term semantic gap to represent the difference between the semantics of two domains as shown in the following figure:




      *Consequences of semantic gap:
  1. Large development times
  2. Large development efforts
  3. Poor quality of software

These issues are tackled by software engineering through the use of methodologies and programming languages(PLs). The software engineering steps aimed at the use of a PL can be grouped into:

  1. Specification, design and coding steps
  2. PL implementation steps

*PL Domain:

Software implementation using a PL domain introduces a new domain, the PL domain. The semantic gap between the application domain and execution domain is bridged by software engineering steps. The first step bridges the gap between Application domain and PL domain, while the second step bridges the gap between the PL domain and execution domain.


Specification Gap:

We refer the gap between application domain and PL domain as the specification-and-design gap or simply the specification gap.


Execution Gap:

The execution gap is bridged by the designer of the programming language processor namely a translator or an interpreter.



Advantages of implementing PL Domain::
  1. The gap to be bridged by the software designer in now between the application domain and PL domain rather than between application domain and execution domain. This reduces the severity of the consequences of the semantic gap mentioned earlier.
  2. Apart from the bridging gap between PL and execution domains the language processor provides capability to detect and indicate errors in the input. This improves the quality of software.

2. Language Processing Activities

We group fundamental language processing activities into two categories as:

  1. Program Generation activities
  2. Program Execution activities

b)Program Execution activities

                A program may be executed through translation or interpretation.



    I) Program Translation:

          The program translation model bridges the execution gap by translating a program written in programming language, called the source program in to equivalent program in the machine or assembly language of the computer system called the target program.




Fig.Program Translation Model


   * Characteristics of Program Translation Model:
  • A program must be translated before it can be executed.
  • A m/c language program generated by translator is saved in file and loaded in main memory and executed whenever desired.( No need to translate every time)
  • Retranslation required after modification in source program.

    II)Program Interpretation:
  • An interpreter executes a program without generating a target program.( line by line).
  • Achieved in 3 different steps:
    1. Fetch the instruction
    2. Decode the instruction
    3. Perform the operation
